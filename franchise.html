Let's commence the description of my project

This project was carried out during your end-of-studies internship and aims to create a franchise management system to centralize and automate the operations of a company's different franchises


Entreprise : Tsunami IT
Project name: Cloud Cooker

1. Technologies and tools used

• Backend: Laravel

• Frontend: Angular

• Architecture: Microservices

• DevOps practices: Continuous integration and continuous deployment (CI/CD), infrastructure monitoring and management

2. Main features

2.1. Franchise management

• Registration and management of franchises: Create, modify, and delete franchises

• Performance monitoring: Monitoring sales, inventory, and financial performance of each franchise

2.2. Product management

• Product catalog: Management of products available in all franchises

• Stock and inventory: Real-time inventory tracking and replenishment management

2.3. Order management

• Order Processing: Centralized order processing system for all franchises

• Delivery tracking: Tracking delivery statuses and managing returns

2.4. User Management

• Authentication and authorization: Secure connection system for administrators, franchise managers, and employees

• Role and permission management: Definition of roles and management of permissions for different users

2.5. Reports and analytics

• Financial Reports: Generate detailed financial reports for each franchise

• Sales Analytics: Analyzes sales data to identify trends and growth opportunities

2.6. Communication

• Notifications: Sending notifications and announcements to franchises

• Internal messaging: Messaging system to facilitate communication between managers and central administration

3. Architecture

3.1. Backend (Laravel)

• RESTful API: Provides endpoints for all system operations

• Microservices: Use of microservices to modularize and manage the different functionalities of the system

• Security: Implementation of JWT for authentication and authorization

3.2. Frontend (Angular)

• Responsive UI: Using Angular components to create a dynamic and responsive user interface

• Communication with the backend: Using Angular services to consume RESTful APIs provided by Laravel

• User experience: Intuitive and easy-to-use design for all types of users (administrators, managers, employees)

3.3. DevOps

• CI/CD: Using continuous integration and continuous deployment pipelines to automate builds, testing and deployments

• Monitoring: Implementation of monitoring tools to ensure system availability and performance

• Containerization: Using Docker to deploy microservices in isolated, consistent containers

4. Benefits for the company

• Centralization of operations: Unified management of franchises, allowing better coordination and efficiency

• Real-time tracking: Real-time performance and inventory monitoring for increased responsiveness

• Automation: Reduction of manual tasks through automation of key processes

• Flexibility: Microservices architecture allowing increased scalability and flexibility

• Security: Protection of sensitive data with advanced security mechanisms



This project aims to modernize and optimize franchise management, providing better visibility, efficiency and coordination for the company.
